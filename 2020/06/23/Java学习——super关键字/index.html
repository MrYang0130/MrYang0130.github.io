Java学习——super关键字

发表于 2020-06-23 更新于 2020-06-28 分类于 -学习

super是指向父类的引用，如果构造方法没有显示地调用父类的构造方法，那么编译器会自动为它加上一个默认的super()方法调用。如果父类由没有默认的无参构造方法，编译器就会报错，super()语句必须是构造方法的第一个子句。

super关键字的学习中遇到了很多问题，上面的引用是我浏览了多篇博客找到的关于super最简洁明了的定义，但别人的学习经验终究不如自己的来的记忆深刻，所以写下这篇笔记来记录学习super的小细节。

1.每个子类构造方法的第一条语句，都是隐含的调用super()，如果父类没有无参构造方法，那么在编译的时候就会报错。

举个例子：

class A
{
public int i;
public A()
{
i = 20;
}
}

class B extends A
{
public int j;
public B(int i, int j)
{
this.j = j;
}
}

public class TestSuper_2
{
public static void main(String[] args)
{
B bb = new B(10, 20);
System.out.printf(“i = %d, j = %d”, bb.i, bb.j);
}
}

结果:

i = 20, j = 20
Process finished with exit code 0

很清晰的可以看到，类B在创建对象的时候，调用了两个参数的构造方法，但只把j的值赋给bb对象的属性，在B的构造方法中没有写super()语句，但仍然使i的值变为20，说明调用了A的构造方法，即隐含的调用了super()。

2.如果显示的写出super()语句，则必须保证该语句是构造方法中的第一条语句，否则编译会报错。

（通俗易懂，不过多解释）

3.super()，如果不写，则编译器会自动添加，如果此时父类没有无参构造方法，那么编译会报错。

例如：

class A
{
public int i;
public A(int i)
{
System.out.printf(“无参构造方法被调用了。\n”);
}
}

class B extends A
{
public int j;
public B(int i, int j)
{
this.j = j;
}
}

public class TestSuper
{
public static void main(String[] args)
{

}

}

此时编译器会标红类B的中构造方法，并提示There is no default constructor available in ‘A’，即类A中没有可供B调用的构造方法。

4.既可以写super()，前提是父类中有无参构造方法；也可以写super(实参)，前提是父类中有对应实参个数的有参构造方法。

（依旧通俗易懂）

5.调用父类构造方法的语句必须借助于super，不能直接写父类的类名（即不能写父类.super，与C++不同）
